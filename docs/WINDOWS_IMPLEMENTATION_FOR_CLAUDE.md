# Задание для Claude: полноценная версия Voice Input для Windows

## Результат

Сделай Windows-дистрибутив этого проекта, которым пользуются как обычной
программой. Пользователь скачивает `VoiceInputSetup.exe`, устанавливает его и
запускает Voice Input через меню «Пуск», ярлык на рабочем столе или трей.
Python, `.py`-файлы, консоль и ручная установка зависимостей пользователю не
видны и не нужны.

Внутри EXE можно оставить текущую Python-реализацию и упакованный interpreter.
Переписывать рабочий STT/UI на другой язык только ради расширения `.exe` не
нужно.

Перед изменениями:

1. Прочитай комментарии в затрагиваемых файлах и существующие тесты.
2. Сделай git tag и полный `git bundle` текущего состояния.
3. Не ломай Linux: все существующие Linux-тесты должны продолжать проходить.

## Какой вариант реализации использовать

Самый простой надёжный путь:

- `PyInstaller --onedir --windowed` создаёт `VoiceInput.exe` и каталог со всеми
  Python/Qt/Whisper DLL. Не используй `--onefile`: ML-зависимости слишком
  большие, запуск медленнее, а ложные срабатывания антивирусов вероятнее.
- Inno Setup 6 упаковывает этот каталог в один `VoiceInputSetup.exe`.
- Сборка выполняется нативно на Windows, лучше через GitHub Actions. Не пытайся
  собирать Windows EXE из Linux.

## Что уже есть и что надо адаптировать

Сохрани текущие QML-интерфейс, Whisper, Silero VAD, настройки, статистику,
перевод речи на английский и словари. В проекте уже есть Windows-ветки для IPC,
глобального hotkey и ввода через `pynput`, но Windows пока не является готовым
продуктом.

Исправь минимально необходимые места:

- Добавь условную зависимость `pynput` только для Windows. `evdev` там не должен
  устанавливаться.
- Вынеси platform paths в маленький общий helper:
  - config: `%APPDATA%\Voice Input\config.json`;
  - logs/cache/models: `%LOCALAPPDATA%\Voice Input\...`;
  - Linux-пути XDG оставь без изменений.
- У frozen-сборки не должно быть обращений к `.venv/bin/python`, `voice-input`,
  shell-командам или Linux desktop integration.
- Ресурсы QML, встроенные шрифты и будущая иконка должны находиться как в source
  checkout, так и внутри `_MEIPASS`/onedir. Сделай один helper пути к ресурсам,
  без разбросанных проверок PyInstaller по проекту.
- На Windows не запускай `notify-send`. Ошибки и состояние показывай через
  `QSystemTrayIcon.showMessage()` и в UI; также пиши rotating log в
  `%LOCALAPPDATA%\Voice Input\logs\voice-input.log`.
- Оставь одну копию приложения. Текущий localhost IPC можно сохранить, если
  команды `--settings`, `--toggle` и `--quit` стабильно доходят до уже
  запущенного EXE. Обработай занятый порт и stale instance без второго демона.
- `on_restart` не должен пытаться выполнить `python -m pill` во frozen-сборке:
  перезапускай `sys.executable` с корректными аргументами.
- Глобальный hotkey должен запускаться без прав администратора, меняться из UI
  и переживать перезапуск.
- Два режима вставки должны иметь реальный смысл:
  - `keyboard`: Unicode-ввод в активное поле;
  - `clipboard`: Win32 clipboard + `Ctrl+V`.
  Для clipboard используй `ctypes`/WinAPI и уже имеющийся keyboard controller,
  не добавляй отдельную тяжёлую библиотеку.
- CPU обязан работать на любом поддерживаемом ПК. Если CUDA DLL отсутствуют,
  `auto` молча откатывается на CPU. Не заявляй GPU-поддержку, пока она не
  проверена на чистой Windows с NVIDIA.
- Модель не вшивай в installer. Первый запуск скачивает выбранную модель в
  пользовательский cache; UI остаётся отзывчивым и явно показывает, что идёт
  загрузка.

## Поведение приложения

Создай один GUI entry point без консольного окна. Обычный запуск поднимает
фоновый процесс и трей. Повторный запуск открывает настройки уже работающего
процесса.

Трей:

- одиночный/двойной клик открывает настройки;
- `Начать / остановить запись`;
- `Настройки`;
- `Выход`;
- tooltip показывает `Voice Input` и текущий hotkey.

Окно остаётся поверх остальных, но не крадёт фокус у поля, куда вставляется
текст. Проверь масштабирование 100%, 125%, 150% и экран 1366×768. Заголовок и
вкладки должны оставаться видимыми, прокручивается только содержимое настроек.

Создай нормальную многоразмерную `voice-input.ico` (16, 24, 32, 48, 64, 128 и
256 px): тёмная стеклянная пилюля и простой янтарный микрофон, без текста.
Используй её для EXE, tray, Start Menu, Desktop и installer.

## Файлы сборки

Добавь отдельный каталог `packaging/windows/`, не смешивая Windows installer с
Linux `install.sh`:

- `voice_input.spec` — явный PyInstaller spec;
- `voice-input.iss` — Inno Setup script;
- `voice-input.ico` и исходник иконки;
- `build.ps1` — одна команда для локальной воспроизводимой сборки;
- при необходимости один небольшой PyInstaller hook для реально пропущенных
  dynamic imports.

`build.ps1` должен:

1. создать чистый venv;
2. поставить зависимости из зафиксированного Windows build-файла;
3. собрать onedir;
4. запустить smoke-test артефакта;
5. вызвать `ISCC.exe` и положить итог в `dist/VoiceInputSetup.exe`.

Не собирай пакеты «на всякий случай». Сначала проверь результат PyInstaller и
добавляй hidden imports/data только по конкретной ошибке. Обязательно включи
QML, шрифты, Qt Quick/Controls, `sounddevice`, `faster_whisper`, `ctranslate2` и
`onnxruntime` со всеми нужными DLL.

## Требования к installer

Установка по умолчанию per-user, без UAC:

- каталог `{localappdata}\Programs\Voice Input`;
- Start Menu shortcut `Voice Input` запускает `VoiceInput.exe --settings`;
- опциональный, включённый по умолчанию Desktop shortcut делает то же;
- опциональный автозапуск, включённый по умолчанию, запускает EXE без окна;
- запись в Apps & Features и штатный uninstaller;
- upgrades поверх предыдущей версии с постоянным `AppId`;
- installer и uninstaller завершают работающий Voice Input корректной командой,
  а не убивают посторонние процессы;
- обычное удаление сохраняет настройки и модели;
- отдельная, не отмеченная по умолчанию опция удаления пользовательских данных;
- архитектура x64, минимальная ОС Windows 10 22H2; Windows 11 обязательно.

Никаких ярлыков на `python.exe`, `.py`, PowerShell или batch-файл. Все ярлыки
указывают только на установленный `VoiceInput.exe` и используют общую ICO.

## CI и релиз

Добавь Windows job в GitHub Actions:

- Windows runner + поддерживаемая стабильная версия Python;
- существующие platform-independent unit tests;
- отдельные Windows-тесты paths, IPC, hotkey formatting и injection command
  generation без реального нажатия клавиш;
- сборка EXE и Inno installer;
- запуск `VoiceInput.exe --self-test` или эквивалентного headless smoke-test;
- загрузка `VoiceInputSetup.exe` как artifact.

Подпись должна быть опциональной: если в secrets есть PFX и пароль — подписать
EXE и installer через `signtool`; если сертификата нет — сборка всё равно
проходит, но artifact явно unsigned. Не коммить сертификаты и пароли.

## Проверка на чистой Windows

Готово только после проверки в Windows Sandbox/чистой VM без Python:

1. Silent install завершается без UAC и ошибок.
2. В Start Menu и на Desktop правильные имя и иконка.
3. После login приложение появляется в tray один раз.
4. Ярлык открывает настройки, второй экземпляр не создаётся.
5. Hotkey начинает и останавливает запись.
6. Русский текст вводится в Notepad; English mode переводит русскую речь на
   английский; clipboard mode вставляет Unicode.
7. Микрофон и список устройств работают; отказ в microphone permission даёт
   понятное сообщение.
8. UI проверен на 1366×768 и scale 100/125/150%.
9. Перезагрузка сохраняет config, hotkey, статистику и model cache.
10. Upgrade сохраняет данные; uninstall удаляет программу и ярлыки, но по
    умолчанию не удаляет модели/настройки.
11. На машине без Python приложение продолжает работать.

В конце дай короткий отчёт: какие файлы изменены, где лежат EXE/Setup, размер
installer, результат чистой VM и что осталось неподписанным или непроверенным.
